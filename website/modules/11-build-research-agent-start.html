📚 Core Concepts

🏗️ Project Structure — Why It Matters

A well-structured agent separates tools, prompts, and agent wiring into their own files. This isn't over-engineering — it's the minimum structure that makes debugging possible.

🔧
tools/ — the agent's hands

search.py, reader.py, writer.py, calculator.py. Each tool is a Python function wrapped with the @tool decorator so the LLM can call it.

📋
prompts/ — the agent's personality

The system prompt defines how the agent behaves: what format to use, how many searches to do, when to stop. Treat it like code — version-control it and iterate.

🤖
agent.py — the wiring

Creates the LLM, connects the tools, and returns a runnable agent. One file, one job — easy to swap out the model or tools later.

▶️
main.py — the entry point

The interactive loop: accepts user input, invokes the agent, and displays tool calls and the final answer. Nothing else lives here.

🔍 Why Tavily for Search?

Your agent needs to search the web. Tavily is purpose-built for agents: it returns clean, structured text that LLMs can read directly. The Google Search API returns HTML designed for human browsers — scripts, ads, navigation — noise the LLM has to filter through before getting to facts.

⚙️ Wiring Up the Agent

Here's the core of agent.py — the moment tools, LLM, and prompt come together:

# agent.py
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langgraph.prebuilt import create_react_agent

from tools.search import search_web
from tools.reader import read_webpage
from tools.writer import save_report
from tools.calculator import calculate
from prompts.system import RESEARCH_AGENT_PROMPT

load_dotenv()

def create_agent():
    llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
    tools = [search_web, read_webpage, save_report, calculate]

    agent = create_react_agent(
        llm,
        tools,
        prompt=RESEARCH_AGENT_PROMPT
    )

    return agent

Key choices: temperature=0 makes tool selection deterministic and consistent. create_react_agent wraps the LLM with the ReAct loop — Think → Act → Observe → Think again — automatically.

⚠️
Tool descriptions are everything. The LLM picks which tool to call based solely on the docstring. A vague description like """Find information.""" leads to wrong choices. Be specific about what the tool does, when to use it, and when NOT to use it.

💬 Watching the Agent Process a Query

Step through how the agent handles a real research request:

User: "What are the top 3 AI agent frameworks in 2024? Compare their pros and cons."
Agent [Thought]: The user wants a current comparison of AI agent frameworks. I need web search — this space changes fast and my training data may be stale.
Agent [Action]: search_web("best AI agent frameworks 2024 comparison LangChain AutoGen LlamaIndex")
Tool Result: [3 results returned — LangChain overview, AutoGen paper summary, LlamaIndex comparison article with URLs and content snippets...]
Agent [Thought]: Good overview. The AutoGen article looks most detailed for pros/cons. Let me read it directly for specifics.
Agent [Action]: read_webpage("https://microsoft.github.io/autogen/docs/getting-started")
Tool Result: [AutoGen docs — multi-agent conversation framework, strong orchestration, steeper learning curve, enterprise focus...]
Agent [Thought]: I now have enough from 2 sources. Time to synthesize into the structured format: Summary → Key Findings → Analysis → Sources → Confidence.
Agent [Answer]: ### Summary — LangChain, AutoGen, and LlamaIndex dominate 2024... [Full structured report with cited sources and confidence: High]

Notice the decisions: The agent chose to search first (not guess), then decided one source needed deeper reading, then stopped when it had enough. That's the ReAct loop — autonomous judgment at each step.

📖 Key Terms

Tavily
A web search API purpose-built for AI agents. Returns clean, structured text instead of raw HTML. Free tier: 1,000 searches/month at tavily.com.
@tool decorator
A LangChain decorator that turns a regular Python function into a callable tool. The function's docstring becomes the tool's description — what the LLM reads to decide when to use it.
create_react_agent
LangGraph function that wires an LLM and a list of tools into the ReAct loop: Reason → Act → Observe → repeat, until the agent produces a final answer.
temperature=0
LLM setting that makes responses deterministic — always picks the most likely token. Agents benefit from this because you want consistent, predictable tool selection across runs.
BeautifulSoup
Python library for parsing HTML. Used in the reader tool to strip scripts, ads, and nav elements — leaving just the readable text content of a webpage.
System Prompt
Instructions given to the LLM before any user message. For agents, this defines behavior: response format, tool usage rules, when to stop. Treat it like code — version-control it.
