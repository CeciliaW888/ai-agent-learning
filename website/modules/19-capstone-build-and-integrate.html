📚 Core Concepts

🛡️ Why Agents Need Error Handling

A multi-agent pipeline has more failure points than a single agent. Any one of these can fail: the LLM call, the web search, the webpage reader, the JSON parsing, the file write. Without error handling, one bad search result crashes the entire pipeline and loses all work done up to that point.

💡
The safe_node() pattern: Wrap each agent node function in a try/except. On failure, return a fallback state with an error message — don't raise. This lets the pipeline continue or terminate cleanly instead of crashing mid-run.

User: python main.py "The future of remote work in 2025" --max-revisions 2 --verbose

Writer Agent [Stage 2/4]: Drafting from research brief. First draft: 920 words. Hook: "In 2019, remote work was a perk. By 2025, it's a negotiation chip." Sending to Editor.
Editor Agent [Stage 3/4 — Revision 0]: Accuracy 8/10, Clarity 6/10, Structure 7/10, Engagement 7/10. Overall: 7/10. Verdict: NEEDS REVISION. 💰 Cost Tracking — Know What You're Spending

Without cost tracking, a single test run can cost more than you expect. A PipelineCostTracker logs input and output tokens per agent so you can see exactly where money is going — and which agent is the most expensive.

⚠️
The revision loop multiplies costs. Each revision round = one more Writer call + one more Editor call. With max 2 revisions and a verbose Editor prompt, a single pipeline run can make 6–8 LLM calls. Track this from day one.

🔄 Isolated Agent vs. Pipeline Agent

🤖 Agent Working in Isolation

Receives input from a human. Returns output to a human. If it fails, the human notices immediately and retries. Error handling is optional — a human is in the loop catching mistakes.

Testing: Simple. Call the function, read the output, judge it yourself.

🔗 Agent in a Pipeline

Receives input from another agent. Returns output to another agent. If it fails silently, the next agent receives garbage — and may produce confident-sounding garbage in return. No human sees the failure until the end.

Testing: You must test the handoffs. What happens if research returns an empty string? What if the editor never writes "APPROVED"?

📖 Key Terms

Graceful Degradation
When something fails, return a safe fallback instead of crashing. The pipeline continues (or exits cleanly) rather than losing all work done up to that point.
Token Cost
LLM APIs charge per token (roughly one word). Input tokens (your prompt) and output tokens (the model's response) have different rates. Longer prompts and longer outputs cost more.
argparse
Python's standard library for parsing command-line arguments. Lets users call your script with flags like --verbose or --max-revisions 3 instead of answering prompts interactively.
Structured Feedback
Editor feedback that specifies exactly what to change and why — not "improve the flow" but "move paragraph 3 before paragraph 2 because the statistics should precede the examples." Structured feedback is what makes revision loops actually improve output.
