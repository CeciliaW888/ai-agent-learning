🗓️ Week 2 Recap — Day by Day

8
LangChain Fundamentals

Set up your environment and built your first chain — input → LLM → output. You learned the framework vocabulary that everything else builds on.

9
Prompt Engineering for Agents

Wrote system prompts that actually shape agent behavior. Discovered that the prompt is as important as the model — the same LLM behaves completely differently with different prompts.

10
Tool Calling Deep Dive

Built production-quality tools with the @tool decorator. Learned why description quality drives tool selection — and wrote descriptions specific enough that the LLM always picks the right one.

11
Building the Research Agent

Combined LangChain, prompts, and tools into a complete working agent. Your first system that can search, synthesize, and save research on any topic.

12
Adding Memory

Added ChromaDB for persistent long-term memory — your agent now recalls past research across sessions. Also added conversation memory so context survives within a session.

13
Testing & Debugging

Added logging, error handling, and verbose tracing. Learned to read agent reasoning traces — and discovered that a tool that fails loudly is much easier to fix than one that fails silently.

📚 How the Pieces Connect

🏗️ The Architecture You Built

Here's everything from Days 8–13 working together in your research agent. Each layer builds on the one below it:

🧠
LangChain → the framework

The plumbing that connects your LLM to tools and memory. Without it you'd be writing all the glue code yourself.

📝
System Prompt → the personality

Defines what the agent is, how it behaves, and when to stop. The same model with different prompts is effectively a different agent.

🔧
Tools → the capabilities

Search, calculate, save. Each tool gives the agent one real-world power. Tool quality matters more than model quality.

💾
Memory → the continuity

ChromaDB for past research, MemorySaver for conversation context. Memory is what makes the agent feel like a colleague rather than a stranger.

🛡️
Debugging → the confidence

Logging, error handling, and tracing. Knowing what your agent is doing — and why — is the difference between something you trust and something you're afraid to show people.

💬 Your Agent Demo

Here's what it looks like to demo your research agent to someone who's never seen it before:

Friend: Wait, you built an AI? What does it actually do?
You: So it's a research agent — you give it a topic, it searches the web, reads multiple sources, and writes you a structured report with citations. Like a research assistant that takes 30 seconds instead of 30 minutes.
Friend: But can't ChatGPT do that?
You: ChatGPT gives you what it memorized in training. My agent actually goes to the web right now, reads current articles, and cites the sources. Also — it remembers what we've researched before and builds on it.
Friend: How does it decide what to search for?
You: I give it a system prompt that tells it how to behave — search first, synthesize second, always cite sources. Then it figures out the right search queries on its own. I also built error handling so if a search fails, it tries again with a different query instead of crashing.
Friend: How long did it take to build?
You: About a week of focused learning — and most of that was understanding the concepts. The actual agent is maybe 150 lines of Python. Want to see it run?
