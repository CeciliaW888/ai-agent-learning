📚 Core Concepts

🗺️ The System Architecture

The pipeline has four specialized roles. Each agent does one thing well and passes its output to the next through a shared state object.

🎯
Coordinator

Receives the user's topic, initializes the shared state, orchestrates the full flow, and delivers the final output. The Coordinator doesn't write content — it manages the process and makes routing decisions at each step.

🔍
Research Agent

Searches the web with 2–3 targeted queries, reads the most promising articles, extracts key facts and statistics, and produces a structured research brief with source citations. Max 3 searches to control API costs.

✍️
Writer Agent

Receives the research brief and produces an 800–1200 word article: hook, clear headers, concrete examples, a strong conclusion. On revision rounds, it receives the editor's feedback and must address all of it explicitly.

📋
Editor Agent

Reviews the draft for accuracy, clarity, structure, and engagement. Scores it 1–10. If the overall score is 8 or above it approves. Below 8 it sends specific, actionable feedback back to the Writer. Auto-approves after 2 revision rounds.

💡
The revision loop: After the Editor reviews, LangGraph evaluates a conditional edge — approved or needs revision? If revision, control flows back to the Writer (not to Research — that would waste API calls and time). Maximum 2 revision rounds, then auto-approve.

💬 The Coordinator in Action

Here's how the Coordinator routes work between sub-agents and tracks the pipeline through to completion:

User: "Write a blog post about AI agents in healthcare."
Coordinator [Plan]: Topic received. Initializing pipeline state. Dispatching to Research Agent with topic + requirements.
Research Agent [Working]: Searching "AI agents healthcare 2024"... reading top 3 articles... extracting key facts and statistics...
Research Agent [Handoff]: Brief ready — 12 key facts, 4 statistics, 6 source URLs. Writing to pipeline state. Passing to Writer.
Writer Agent [Working]: Crafting article from research brief... hook about misdiagnosis rates... 4 sections... conclusion... 1,050 words.
Writer Agent [Handoff]: Draft complete. 1,050 words. Sending to Editor for review.
Editor Agent [Reviewing]: Accuracy 9/10, Clarity 8/10, Structure 8/10, Engagement 8/10. Overall: 8/10. Verdict: APPROVED.
Coordinator [Done]: Pipeline complete. 1,050 words | 6 sources | 0 revisions needed | Saved to output/ai-agents-in-healthcare.md

📁 Project Structure

Proper separation of concerns makes each piece independently testable and replaceable. Here's what you're building today:

📄
pipeline.py — The wiring

Defines PipelineState, all LangGraph nodes, edges, and the conditional routing logic. The Coordinator lives here — but as graph structure, not an agent.

📂
agents/ — The workers

researcher.py, writer.py, editor.py — each exports one node function. No agent imports from another agent. They only read and write shared state.

🔧
tools/ — The capabilities

search.py, reader.py, file_ops.py — shared tools any agent can use. Keeping tools separate means you can swap Tavily for a different search provider without touching any agent code.

💬
prompts/ — The instructions

System prompts stored as separate Python strings. This lets you tune agent behavior — making the Editor stricter or the Writer more concise — without touching any logic code.

⚠️
Design the state before writing agents. The PipelineState TypedDict is the contract between your agents. If you don't define it clearly first, you'll spend hours debugging mismatched dictionary keys between nodes. State first, agents second.

🔄 Single Agent vs. Multi-Agent Pipeline

🔍 Week 2: Single Research Agent

One agent does everything — searches, reads, synthesizes, and writes. Simple to build and debug. Works well for focused research summaries.

The limitation: When you ask it to research thoroughly and write engagingly and fact-check itself, quality drops across all three. One prompt can't optimize for everything at once.

🏭 Week 3: Multi-Agent Pipeline

Each agent is a specialist with its own focused prompt. The Researcher only researches. The Writer only writes. The Editor only edits. Each is independently tunable.

The advantage: Specialization + quality gates. The Editor catches what the Writer missed. Revision loops create an improvement cycle. Much harder to build — but output quality jumps significantly for complex tasks.

📖 Key Terms

Coordinator / Supervisor
The orchestrating agent that doesn't do content work itself — it routes tasks, manages state, and decides what happens next based on what sub-agents return.
Pipeline State
A shared TypedDict all agents read from and write to. Contains everything: topic, research brief, draft, editor feedback, revision count, and final article.
Conditional Edge
A LangGraph routing decision. After the Editor node, a function reads the approved field and routes to either "finalize" or back to "write" for a revision round.
Revision Loop
When the Editor rejects a draft, the pipeline cycles back to the Writer (not Research). Each cycle increments revision_count. A max revision limit prevents infinite loops.
Separation of Concerns
Each file does exactly one thing. Agents don't import from other agents. Tools are shared libraries. Prompts are separate from logic. This makes every piece independently testable.
