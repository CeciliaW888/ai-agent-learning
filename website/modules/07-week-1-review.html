📋 Week 1 Recap — One Concept Per Day

1
Day 1: What is an AI Agent?

An agent is an autonomous system that perceives its environment, decides what to do, and acts — repeatedly — until a goal is achieved. The key word is autonomous: it keeps going without you guiding every step.

2
Day 2: The PDA Loop

Perceive → Decide → Act → Repeat. This loop is the heartbeat of every agent system. Each iteration, the agent takes in new information, reasons about what to do next, and executes an action. The loop runs until the goal is met or a limit is hit.

3
Day 3: Agent vs Chatbot vs RAG

Chatbots respond once. RAG retrieves then responds once. Agents act in a loop with tools. Don't reach for an agent when a chatbot will do — start simple. Use agents when the task requires multiple steps, real-time data, or taking real actions.

4
Day 4: Tools & Function Calling

LLMs don't execute code — they output JSON describing what they'd like to call. Your code actually calls the function and returns the result. This is a crucial architecture detail: the LLM decides, your system acts.

5
Day 5: Memory & State

Agents have three memory types: short-term (current conversation), long-term (vector DB across sessions), and working memory (scratchpad for the current task). Memory is what turns a one-shot LLM into an agent that learns.

6
Day 6: Planning & Reasoning

CoT (think step by step), ReAct (interleave thought + tool use), ToT (explore multiple paths). Different problems call for different strategies. The secret weapon: task decomposition — break big goals into small, manageable steps.

🔗 How They All Connect

The six concepts from this week aren't six separate topics — they're one system:

💡
The unified model: An AI Agent (Day 1) is an LLM that runs in a PDA Loop (Day 2), using Tools (Day 4) to act on the world, Memory (Day 5) to learn and persist state, and Planning (Day 6) to break complex goals into steps. This whole system only makes sense once you know when to reach for it — not for simple Q&A (chatbot) or knowledge retrieval (RAG), but for multi-step tasks that require acting in the world (Day 3).

💬 Ask the Agent — Review Session

Here's a simulated Q&A session — the kind of review dialogue you might have with an AI tutor:

Student: "I keep mixing up CoT and ReAct. What's the actual difference?"
Tutor: "Good question. Chain of Thought is all internal — the LLM reasons step by step using only what it already knows. No tools, no external data."
Tutor: "ReAct interleaves thought with action. After each reasoning step, it can call a tool — search the web, read a file, run code — then observe the result before thinking again."
Student: "So ReAct is CoT... but with tool calls in between the thoughts?"
Tutor: "Exactly. ReAct = CoT + Tool Use, interleaved. The tool results become new observations that feed back into the next thought. That's what 'grounds' the reasoning in real data instead of hallucination."
Student: "And what about the PDA loop — where does that fit in?"
Tutor: "ReAct IS the PDA loop at the reasoning level. Each Thought is the 'Decide' phase. Each Action is the 'Act' phase. Each Observation is the 'Perceive' phase for the next iteration. They're the same pattern at different levels of abstraction."
Student: "That actually just made everything click. Thank you."

📖 Week 1 Terms — Full Glossary

AI Agent
An autonomous system that perceives, decides, and acts in a loop to achieve a goal. Requires tools, memory, and planning layered around an LLM.
PDA Loop
Perceive → Decide → Act → Repeat. The fundamental operating cycle of every agent system. The loop runs until the goal is achieved or a safety limit is hit.
RAG
Retrieval-Augmented Generation — retrieve relevant documents, then generate a response. Not an agent: one retrieval, one response. Best for knowledge-base Q&A.
Function Calling
The mechanism where an LLM outputs a JSON description of a tool call, and your code executes the actual function. The LLM decides; your code acts.
Vector Database
A database that stores information as embeddings (numerical vectors) and retrieves by semantic similarity. The backbone of long-term agent memory.
ReAct
Reasoning + Acting — the pattern where an agent interleaves Thought → Action → Observation steps. Grounds reasoning in real data by calling tools between each thinking step.
Chain of Thought (CoT)
Prompting the LLM to reason step by step before answering. Improves accuracy on complex problems. Unlike ReAct, CoT uses no external tools.
Task Decomposition
Breaking a complex goal into smaller, ordered sub-tasks. The core planning technique that makes large goals tractable for agents.
