📚 Core Concepts

📬 Three Ways Agents Talk to Each Other

1
Shared State

All agents read and write to a common data structure — like a shared whiteboard. Agent A adds research notes; Agent B reads them and writes a draft; Agent C reads the draft and adds feedback. Simple, visible, but the state can grow large and agents can accidentally overwrite each other.

2
Message Passing

Agents send discrete messages directly to each other — like email. Each message has a sender, receiver, content, and type (request / response / handoff). More structured and private than shared state, but requires a message bus or router to coordinate delivery.

3
Handoff (Transfer of Control)

One agent explicitly passes control — and its full work product — to the next agent. Like handing off a baton in a relay race. The key rule: include enough context so the receiving agent doesn't need to ask what's going on. Never assume the next agent knows what you were thinking.

💡
Common gotcha: When Agent A passes to Agent B, make sure enough context is included. Agents don't share memory by default. If Agent B doesn't understand the full situation, it will make decisions based on incomplete information — and the errors compound downstream.

💬 A Structured Handoff in Action

Watch how two agents pass a task with a structured handoff message:

User → Researcher: "Research the latest developments in AI coding assistants."
Researcher [Working]: Searching web, reading articles, extracting key facts...
Researcher [Action]: transfer_to_writer({ "task": "Write a 500-word article", "research_summary": "Key findings: GitHub Copilot 70% adoption, Cursor editor rising, Claude and GPT-4 powering most tools, debate over code quality...", "target_audience": "developers", "tone": "informative" })
[HANDOFF] Researcher → Writer. Context received: task, research, audience, tone.
Writer [Thought]: I have everything I need. Research summary is clear, audience is defined. I can start writing.
Writer [Action]: write_article(research_summary, audience="developers", word_count=500)
Writer → User: "Article complete. 498 words, 4 sections covering adoption trends, top tools, tradeoffs, and what's next."

Notice the handoff message: it's structured — task, research, audience, tone. The Writer gets everything it needs without having to ask. That's a good handoff.

⚠️ Common Pitfalls in Agent Communication

⚠️
Watch out for infinite revision loops. Writer → Editor → Writer → Editor → ... Set a maximum revision count in your state. If you don't, agents will happily loop forever, burning tokens and time.

🔄 Shared State vs Message Passing

🗂️ Shared State

All agents read and write a common data structure. Like a shared whiteboard everyone can see.

Use when: agents need to see the full picture, the workflow is well-defined, and you want simple debugging (just inspect the state object).

Watch out: state grows large; agents can accidentally clobber each other's data.

📨 Message Passing

Agents send discrete messages directly to each other. Each message has a clear sender, receiver, and type.

Use when: agents need privacy from each other, communication is event-driven, or you want to decouple agents so they can run independently.

Watch out: needs a message bus to coordinate; context can get lost if messages aren't well-structured.

📖 Key Terms

Shared State
A common data structure all agents can read and write — like a shared whiteboard. Simple to inspect but requires careful coordination to avoid conflicts.
Message Passing
Agents send structured messages directly to each other. Each message has a sender, receiver, content, and type (request / response / handoff).
Handoff
One agent explicitly transfers control and context to another. A good handoff includes enough information so the next agent can act without asking follow-up questions.
State Graph
A graph structure (like LangGraph) that defines agent nodes and the edges between them — including conditional edges that route based on current state.
Conditional Edge
A transition in a state graph that routes to different nodes depending on the state — e.g., "if the editor says APPROVED, end; otherwise, send back for revision."
Message Bus
A central hub that routes messages between agents. Agents register with the bus and send messages to named recipients — like a postal service for agents.
