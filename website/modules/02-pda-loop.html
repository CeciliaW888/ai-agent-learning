📚 Core Concepts

🔍 Phase 1: Perceive

Before an agent can do anything, it needs to understand its situation. The Perceive phase is when the agent takes in everything available to it:

📥
User input

The initial request or follow-up message. E.g. "Find me the best Python course under $50."

🔧
Tool results

Output from the last action — search results, code output, database records.

🧠
Memory

What the agent already knows — past conversation, saved notes, retrieved documents.

💡
Key insight: Everything an agent "sees" fits inside its context window — the LLM's working memory. The art of agent design is deciding what to put in that window at each step.

🤔 Phase 2: Decide

This is where the LLM does its work. Given everything it perceives, it reasons about what to do next:

Should I search the web? Run some code? Ask the user for clarification? Or do I have enough to give a final answer?

The ReAct pattern makes this explicit by prompting the agent to write out its reasoning (Thought:) before choosing an action (Action:). You'll implement this in Week 2.

⚡ Phase 3: Act

The agent executes whatever it decided — then the result feeds back into the next Perceive phase. Common actions:

🌐
Tool call

search_web(), run_python(), query_database(), send_email()...

💬
Final answer

The agent decides it has enough information and responds. The loop ends.

❓
Clarification request

Asks the user for more information before continuing.

🔄 Watching the Full Loop in Action

Here's a real PDA loop traced step by step: "What's Apple's stock price and is it higher than last week?"

[Loop 1 — Perceive] Input: "What's Apple's stock price and is it higher than last week?"
[Loop 1 — Decide] Thought: I need live data. I'll call the stock tool for today's price.
[Loop 1 — Act] Action: get_stock_price("AAPL", period="today")
[Loop 2 — Perceive] Observation: AAPL today = $213.45
[Loop 2 — Decide] Thought: I have today's price but still need last week's. One more call.
[Loop 2 — Act] Action: get_stock_price("AAPL", period="7d_ago")
[Loop 3 — Perceive] Observation: AAPL last week = $208.12
[Loop 3 — Decide] Thought: $213.45 > $208.12. I have enough to answer now.
[Loop 3 — Act] Answer: "Apple is $213.45 today, up 2.6% from $208.12 last week."

Three loops, two tool calls, one answer. The agent only stopped when it had everything it needed.

⚠️
Infinite loops are real: An agent that never decides "I have enough to answer" will loop forever — burning tokens and money. Good agent design always includes a clear stopping condition. You'll handle this in Week 2.

📖 Key Terms

PDA Loop
Perceive → Decide → Act. The three-phase cycle every agent runs repeatedly until a task is complete.
ReAct (Reason + Act)
A prompting pattern where the agent writes explicit Thought → Action → Observation steps. Makes reasoning transparent and debuggable.
Context Window
The agent's working memory — everything it can "see" at once. The Perceive phase is about deciding what goes in here.
Observation
The result of an action — search results, code output, API responses. Feeds directly into the next Perceive phase.
Stopping Condition
The rule that ends the loop — "I have enough to answer" or "max steps reached." Critical for preventing infinite loops.
