🎯 Today's Polish Goals

Every agent node and tool call should catch exceptions and return structured errors — no silent failures, no bare stack traces in production output.

2

Log every agent decision, tool call, and pipeline state transition to a file. After any run, you should be able to reconstruct exactly what happened and why.

3
Build a CLI interface

4

Run the full pipeline on a tech topic, a business topic, and a lifestyle topic. Fix every edge case you discover — this is where hidden bugs surface.

5
Write the architecture README

Document each agent, the data flow between them, setup instructions, and usage examples. If a friend can clone and run it in 10 minutes, you're done.

📚 Core Concepts

🏭 Prototype vs Production-Ready

The code you wrote on Days 18–19 works — but "works on my machine" is a prototype. Production-ready means it works reliably for other people, in conditions you didn't test, without you there to explain it.

✅
Production-ready agent code has:

Consistent error handling across all nodes, structured logging you can search after the fact, clear CLI interface, .env.example so others can set up API keys, a README with a working quick-start, and at least basic tests.

🚧
Prototype code has:

Debug print statements scattered throughout, hardcoded API keys or paths, no tests, no README, and it works only if you run exactly the right command in exactly the right directory.

💡
Logging best practice for agents: Log at the pipeline boundary (topic in → article out), at each agent handoff (researcher → writer, writer → editor), and inside every tool call. Use logging.INFO for normal flow and logging.WARNING for fallbacks. Avoid logging the full LLM prompt/response to files — that's token-heavy and can contain sensitive data.

⌨️ CLI Setup with argparse

Here's a clean CLI setup for your content pipeline. Notice how it handles defaults gracefully and prints usage when arguments are wrong:

"""
main.py — Multi-Agent Content Pipeline CLI
Usage: python main.py "Your topic here" [options]
"""
import argparse
import logging
import sys
from pipeline import build_pipeline, PipelineState

# Set up file + console logging
logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(message)s",
    handlers=[
        logging.FileHandler("pipeline.log"),
        logging.StreamHandler()
    ]
)
logger = logging.getLogger("pipeline")


def parse_args():
    parser = argparse.ArgumentParser(
        description="Multi-agent content pipeline: research → write → edit",
        formatter_class=argparse.RawDescriptionHelpFormatter,
        epilog="""
Examples:
  python main.py "The future of AI in healthcare"
  python main.py "Remote work trends" --requirements "Focus on data"
  python main.py "Python vs JavaScript" --max-revisions 3 --verbose
        """
    )
    parser.add_argument(
        "topic",
        help="The topic to research and write about"
    )
    parser.add_argument(
        "--requirements",
        default="",
        help="Additional requirements for the article (optional)"
    )
    parser.add_argument(
        "--max-revisions",
        type=int,
        default=2,
        help="Maximum editor revision cycles (default: 2)"
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Print agent reasoning steps as they happen"
    )
    return parser.parse_args()


def main():
    args = parse_args()

    logger.info(f"Starting pipeline for topic: '{args.topic}'")

    pipeline = build_pipeline(max_revisions=args.max_revisions)

    initial_state: PipelineState = {
        "topic": args.topic,
        "requirements": args.requirements,
        "research": "",
        "sources": [],
        "draft": "",
        "feedback": "",
        "approved": False,
        "revision_count": 0,
        "final_article": "",
        "metadata": {}
    }

    try:
        result = pipeline.invoke(initial_state)
        article = result.get("final_article", "")

        if not article:
            logger.error("Pipeline completed but no article was produced")
            sys.exit(1)

        print("\n" + "=" * 60)
        print(f"TOPIC: {args.topic}")
        print("=" * 60)
        print(article)
        print("=" * 60)

        meta = result.get("metadata", {})
        print(f"\nWord count: {meta.get('word_count', 'unknown')}")
        print(f"Revisions: {result.get('revision_count', 0)}")
        logger.info("Pipeline complete.")

    except Exception as e:
        logger.error(f"Pipeline failed: {e}")
        sys.exit(1)


if __name__ == "__main__":
    main()

📖 Key Terms

argparse
Python's standard CLI argument parsing library. Turns your script into a proper command-line tool with help text and argument validation.
.env.example
A template showing what environment variables are needed, with placeholder values. Lets others set up API keys without exposing yours.
Architecture Doc
A README section describing each agent, the data flow between them, and the overall system design. Essential for anyone (including future you) who reads the code.
Structured Logging
Logs with consistent format (timestamp, level, message) written to a file. Makes post-run debugging possible — you can reconstruct exactly what happened.
