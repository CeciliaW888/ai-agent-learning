📚 Core Concepts

🗂️ Three Layers of Agent Memory

Memory isn't one thing — it has layers, just like human memory. Today you'll build all three:

1
Conversation Memory — within a session

"What did we just discuss?" — LangGraph's MemorySaver handles this automatically using a thread ID. Same thread = same conversation context.

2
Research Memory — across sessions

"What did I research last week?" — ChromaDB stores findings as vector embeddings on disk, searchable by meaning across restarts.

3
User Preferences — permanent

"I prefer concise bullet points." — A simple JSON file remembers working style and preferences that should survive forever.

🔗 Conversation Memory with MemorySaver

LangGraph's MemorySaver persists the conversation state. The thread_id is the key — same thread means the agent remembers:

# Adding conversation memory to your agent
from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.memory import MemorySaver

memory = MemorySaver()

agent = create_react_agent(
    llm,
    tools,
    checkpointer=memory  # This enables conversation memory
)

# Use the same thread_id to keep context alive
config = {"configurable": {"thread_id": "research-session-1"}}

result1 = agent.invoke(
    {"messages": [{"role": "user", "content": "Research AI agents"}]},
    config=config
)

# The agent remembers the first exchange
result2 = agent.invoke(
    {"messages": [{"role": "user", "content": "Compare that with traditional chatbots"}]},
    config=config  # Same thread_id = same conversation
)
💡
The thread_id is like a conversation ID. Same thread = agent remembers. Different thread = fresh start. You can have multiple parallel conversations by using different thread IDs.

💾 Long-Term Memory with ChromaDB

Conversation memory disappears when you restart. ChromaDB persists research findings to disk so the agent can recall them in future sessions:

# tools/memory.py — save and recall research findings
import chromadb
from datetime import datetime
from langchain_core.tools import tool

# Persists to disk — survives restarts
chroma_client = chromadb.PersistentClient(path="./memory_db")
research_collection = chroma_client.get_or_create_collection(
    name="research_findings"
)

@tool
def save_to_memory(topic: str, findings: str) -> str:
    """Save research findings to long-term memory for future reference.

    Use this AFTER completing research on a topic.
    These findings will be available in future conversations.

    Args:
        topic: The research topic (e.g., "AI agent market size 2024")
        findings: Key findings to remember — keep concise and factual
    """
    research_collection.add(
        documents=[findings],
        metadatas=[{"topic": topic, "date": datetime.now().isoformat()}],
        ids=[f"research_{research_collection.count()}"]
    )
    return f"Saved to memory: '{topic}' ({len(findings)} chars)"

@tool
def recall_research(query: str) -> str:
    """Search memory for past research related to a query.

    Call this BEFORE searching the web — you may already have the answer.

    Args:
        query: What you are looking for (e.g., "AI market size")
    """
    if research_collection.count() == 0:
        return "No past research found. This is a fresh start."

    results = research_collection.query(query_texts=[query], n_results=3)

    if not results["documents"][0]:
        return f"No relevant past research found for '{query}'."

    memories = []
    for doc, meta in zip(results["documents"][0], results["metadatas"][0]):
        memories.append(
            f"Topic: {meta.get('topic', 'Unknown')} "
            f"({meta.get('date', '')[:10]})\n{doc[:300]}"
        )
    return f"Found {len(memories)} past findings:\n\n" + "\n\n".join(memories)

💬 Memory Across Three Conversation Turns

Watch the agent use memory — checking past research before searching, then saving new findings:

Turn 1 — User: "Research the latest AI agent frameworks and save your findings."
Agent [Action]: recall_research("AI agent frameworks") → "No past research found. Fresh start."
Agent [Action]: search_web("top AI agent frameworks 2024 LangChain AutoGen LlamaIndex")
Agent [Action]: save_to_memory("AI agent frameworks 2024", "LangChain: most popular, large ecosystem... AutoGen: Microsoft, multi-agent focus... LlamaIndex: data-heavy RAG...")
Agent: "Here's a summary of the top frameworks... [structured report]. Findings saved to memory."
Turn 2 — User: "Which of those would you recommend for a beginner building their first agent?"
Agent [Thought]: The user is following up on the frameworks I just researched. I remember the conversation context — I don't need to search again.
Agent: "Based on what we just covered, I'd recommend LangChain for beginners — largest community, most tutorials, and the ecosystem you've been using in this course."
Turn 3 — (New session, next day) — User: "What do you know about AI agent frameworks?"
Agent [Action]: recall_research("AI agent frameworks") → "Found 1 past finding: LangChain, AutoGen, LlamaIndex... [the findings saved yesterday]"
Agent: "Based on my previous research from [date]: LangChain is most popular for beginners, AutoGen suits multi-agent orchestration, LlamaIndex excels at data-heavy RAG tasks."

Three types of memory at work: Turn 2 used conversation memory (same session). Turn 3 used long-term research memory (new session, ChromaDB recall).

🆚 Stateless vs Stateful Agent

📋 Stateless Agent (no memory)

Every invocation is independent. The agent has no idea what was asked before — you have to repeat context every time.

"What did we discuss yesterday?" → Agent: "I don't have any record of previous conversations."

Simple to build. Cheap to run. Frustrating to use for multi-step work.

← Tap to flip →

🧠 Stateful Agent (with memory)

Remembers the conversation, past research, and user preferences. Can build on prior work instead of starting from scratch every time.

"What did we discuss yesterday?" → Agent: "Yesterday we researched quantum computing — here are the key findings I saved..."

More powerful. Requires thoughtful memory strategy — too much memory creates noise.

⚠️
Don't store everything. Saving every search query creates noise that pollutes future recalls. Save key findings, decisions, and preferences — not working notes or intermediate steps. Less is more in agent memory.

📖 Key Terms

MemorySaver
LangGraph's built-in checkpointer that persists conversation state in memory. Tied to a thread_id — same ID means the agent continues the conversation rather than starting fresh.
Thread ID
A unique identifier for a conversation session. Same thread_id across multiple agent.invoke() calls = continuous conversation with shared context.
ChromaDB
Open-source vector database that stores text as embeddings and supports semantic search. PersistentClient mode saves data to disk — findings survive restarts.
Checkpointer
LangGraph's mechanism for persisting state between agent runs. MemorySaver is the in-memory version; you can swap it for a database-backed version in production.
Memory Protocol
Rules defined in the system prompt for when to save and recall memories. Example: "Before searching, recall_research(). After completing research, save_to_memory()."
Memory Pruning
The practice of removing outdated or irrelevant memories. Important because stale data (last year's market size) can mislead the agent just as much as no data.
