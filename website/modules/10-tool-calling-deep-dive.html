📚 Core Concepts

🏗️ How Tool Calling Works

When you give an LLM a list of tools, it doesn't see the Python code — it sees a JSON schema for each tool: name, description, and parameter types. The LLM reads this schema and decides which tool to call and with what arguments. The framework runs the actual Python function, returns the result as an observation, and the LLM continues reasoning from there.

1
LLM reads the schema

Tool descriptions are injected into the system prompt. The LLM learns what tools exist, what they do, and what arguments they need.

2
LLM decides: which tool + what args

Based on the request and descriptions, the LLM outputs a structured tool call — e.g. search_web("AI agent frameworks 2024").

3
Framework executes, returns observation

The framework calls your Python function and passes the result back to the LLM as an "observation." The agent reads it and decides what to do next.

4
Loop until done

The agent keeps calling tools and reading observations until it has enough to give a final answer. This is the ReAct loop you learned in Week 1.

💡
The description field is the most important part. The LLM reads your tool's docstring to decide when to use it. A vague description like "find information" leads to wrong tool selection. A specific description like "Search the web for current facts. Use ONLY for real-time information. Do NOT use for math calculations." guides the LLM precisely — every time.

📝 Defining Two Tools with @tool

Here's how to define a search tool and a calculator in LangChain. Notice how the docstrings are written for the LLM — they explicitly say when to use each tool and what to avoid:

from langchain_core.tools import tool
import os

@tool
def search_web(query: str) -> str:
    """Search the internet for current, real-world information.

    Use this for: facts, news, recent events, statistics,
    anything that requires up-to-date data from the web.
    Do NOT use for: math, arithmetic, or calculations —
    use the calculate tool instead.

    Args:
        query: A specific search query. More specific = better results.
               Example: "AI agent frameworks Python 2024 comparison"
    """
    try:
        from tavily import TavilyClient
        client = TavilyClient(api_key=os.getenv("TAVILY_API_KEY"))
        results = client.search(query, max_results=3)

        if not results.get("results"):
            return f"No results found for '{query}'. Try rephrasing."

        formatted = []
        for r in results["results"]:
            formatted.append(
                f"**{r['title']}**\n"
                f"{r['content'][:300]}\n"
                f"Source: {r['url']}"
            )
        return "\n\n---\n\n".join(formatted)
    except Exception as e:
        return f"Search error: {str(e)}"


@tool
def calculate(expression: str) -> str:
    """Evaluate a math expression. Use for ANY arithmetic,
    percentages, ratios, or numeric calculations.

    Never estimate — always use this tool when math is involved.
    Do NOT use for looking up facts or searching the web.

    Args:
        expression: A valid math expression using numbers and +-*/()
                    Example: "100 * 0.15" to get 15% of 100
                    Example: "(50 + 30) / 2" for an average
    """
    safe_chars = set("0123456789+-*/.() ")
    if not all(c in safe_chars for c in expression):
        return "Error: Use only numbers and +-*/(). No variables or text."
    try:
        result = eval(expression)
        return f"{expression} = {result}"
    except ZeroDivisionError:
        return "Error: Division by zero."
    except Exception:
        return f"Error: Could not evaluate '{expression}'. Check syntax."

💬 Watching the Agent Choose

Here's an agent deciding which tool to use — notice how the descriptions drive the decision:

User: What's the current market cap of Nvidia?
Agent [Thought]: This is a real-time fact — I need current data from the web. The search_web tool is for exactly this. I should not estimate or calculate this.
Agent [Action]: search_web("Nvidia market cap current 2024")
Tool Result: Nvidia's market cap reached approximately $3.3 trillion in late 2024, making it one of the most valuable companies in history. Source: Reuters
Agent: Nvidia's market cap is approximately $3.3 trillion as of late 2024.
User: If I'd invested $5,000 when it was at $1 trillion, what would it be worth now?
Agent [Thought]: This is pure arithmetic — $1T to $3.3T is a 3.3x multiplier. I should use the calculate tool, not search. The description says to never estimate math.
Agent [Action]: calculate("5000 * 3.3")
Agent: Your $5,000 would be worth approximately $16,500 today — a 3.3x return based on Nvidia's growth from $1T to $3.3T market cap.

The key: The agent used search_web for a real-time fact and calculate for arithmetic — because the descriptions made those boundaries unambiguous. This is why description quality matters more than almost anything else.

📄 Schema vs Execution

📄 Tool Definition (Schema)

What the LLM sees — the name, description, and parameter types injected into the system prompt. This is what determines whether the agent picks the right tool.

"search_web: Search the internet for current, real-world information. Use for facts and news. Do NOT use for math."

The description is the instruction manual — written for the model, not for Python.

📖 Key Terms

@tool decorator
LangChain decorator that turns a Python function into an agent tool. The function's docstring becomes the description the LLM reads to decide when to use it.
JSON Schema
The structured description of a tool — name, description, parameters, and types. This is what gets injected into the LLM's context window.
Observation
The result the LLM receives after a tool call. The agent reads this to decide its next action — search again, calculate, or give a final answer.
Single Responsibility
Each tool should do exactly one thing well. "Search + summarize + save" should be three separate tools — cleaner descriptions, better selection accuracy.
Tool Chaining
When an agent calls multiple tools in sequence — search → calculate → save — to complete a task that no single tool could handle alone.
Tavily
A search API purpose-built for AI agents — returns clean, structured results optimized for LLM consumption, not rendered HTML.
